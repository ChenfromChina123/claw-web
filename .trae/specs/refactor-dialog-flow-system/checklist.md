# Checklist

## 后端核心功能验证

- [x] Agent Loop 能够支持多轮工具调用（至少测试3轮连续调用）
- [x] 每个工具调用都发送完整的4个事件（use/start/end 或 use/start/error）
- [x] 工具调用成功时，前端能收到并显示结果
- [x] 工具调用失败时，前端能收到并显示错误信息
- [x] 循环在AI生成纯文本回复时正确终止
- [x] 循环在达到最大迭代次数（10次）时正确终止并发送警告
- [x] 单个工具调用超时时能捕获异常并继续下一轮

## 前端状态同步验证

- [x] toolCalls 数组在工具执行过程中实时更新
- [x] 工具状态流转正确：pending → executing → completed/error
- [x] UI 能正确展示不同状态的视觉样式（颜色、图标、标签文本）
- [x] 错误信息对用户友好（显示可操作的建议而非技术细节）
- [x] 多个工具调用按时间顺序排列，序号正确

## 边界情况和容错验证

- [x] WebSocket 断开重连后，工具状态能恢复或清空（不残留僵尸状态）
- [x] 用户快速发送多条消息时，工具调用不混乱（每个消息的循环独立）
- [x] AI 返回空响应或异常响应时，系统不崩溃
- [x] 工具输入参数缺失或格式错误时，有明确的校验错误提示

## 性能和用户体验验证

- [x] 流式文本输出流畅，无明显卡顿
- [x] 工具执行期间界面保持响应（不阻塞用户操作）
- [x] 长时间运行的工具（>5秒）有进度指示
- [x] 控制台日志清晰，便于排查问题（包含sessionId、iteration、toolName等关键信息）
