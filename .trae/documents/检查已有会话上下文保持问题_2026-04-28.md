# 问题分析：App端AI没有上下文

## 问题描述
用户在手机App端的已有会话中发送消息时，AI感觉没有上下文，无法继续之前的对话。

## 根本原因

### 问题代码位置
[sessionManager.ts:132](file:///d:/Users/Administrator/AistudyProject/claw-web/server/src/master/services/sessionManager.ts#L132)

```typescript
// 过滤掉内部的 tool_result 消息（这些是工具调用的返回值，不应显示给用户）
const visibleMessages = this.filterVisibleMessages(dbMessages)
```

### 问题分析

`filterVisibleMessages` 方法会过滤掉 `tool_result` 类型的消息：

```typescript
private filterVisibleMessages(messages: Message[]): Message[] {
  return messages.filter(msg => {
    if (Array.isArray(msg.content)) {
      const hasToolResult = msg.content.some(
        (block: any) => block && block.type === 'tool_result'
      )
      if (hasToolResult) {
        return false  // 过滤掉 tool_result 消息
      }
    }
    return true
  })
}
```

### 为什么这会导致上下文丢失？

1. **AI对话流程**：
   - 用户发送消息 → AI回复（可能包含 `tool_use`）
   - 工具执行 → 结果以 `tool_result` 消息存储
   - AI收到 `tool_result` → 基于结果继续回复

2. **`tool_result` 消息的作用**：
   - 存储工具执行的返回结果
   - 是AI理解之前发生了什么的关键上下文
   - 没有它，AI就不知道之前执行了什么工具、返回了什么结果

3. **问题触发条件**：
   - App端每次进入会话都会调用 `loadSession()` 从数据库加载
   - `loadSession()` 会过滤掉 `tool_result` 消息
   - 导致传给AI的消息列表缺少关键的上下文信息

### 为什么Web端可能没有这个问题？

Web端的会话可能一直保持在内存中，不需要从数据库重新加载。但如果Web端也会从数据库加载会话（比如刷新页面），应该也会遇到同样的问题。

## 解决方案

### 方案1：不过滤 `tool_result` 消息（推荐）

修改 `loadSession()` 方法，保留所有消息用于AI上下文：

```typescript
async loadSession(sessionId: string): Promise<InMemorySession | null> {
  const dbMessages = await this.messageRepo.findBySessionId(sessionId)
  
  // 不再过滤 tool_result 消息，因为它们是AI上下文的一部分
  // const visibleMessages = this.filterVisibleMessages(dbMessages)
  const messages = dbMessages  // 保留所有消息
  
  // ... 其余代码
}
```

**优点**：简单直接，确保AI获得完整上下文
**缺点**：前端需要自己过滤显示

### 方案2：分离显示消息和AI消息

在 `InMemorySession` 中维护两个消息列表：
- `messages`: 完整消息列表（包含 `tool_result`），用于AI上下文
- `visibleMessages`: 过滤后的消息列表，用于前端显示

```typescript
interface InMemorySession {
  session: Session
  messages: Message[]        // 完整消息，用于AI
  visibleMessages: Message[] // 过滤后消息，用于显示
  toolCalls: ToolCall[]
  dirty: boolean
}
```

### 方案3：在调用AI时重新加载完整消息

在 `callAIWithStream()` 中，从数据库重新加载完整的消息列表：

```typescript
private async callAIWithStream(...) {
  // 从数据库加载完整消息（包括 tool_result）
  const dbMessages = await this.messageRepo.findBySessionId(sessionId)
  const messages = dbMessages  // 不过滤
  // ...
}
```

## 推荐方案

**方案1** 是最简单直接的解决方案。`tool_result` 消息虽然不应该显示给用户，但它们是AI上下文的重要组成部分，不应该在加载会话时被过滤掉。

前端应该负责过滤显示的消息，而不是在后端加载时过滤。
