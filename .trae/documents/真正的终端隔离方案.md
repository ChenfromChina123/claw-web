# 真正的终端隔离方案 - Linux Namespaces 实现

## 问题分析

当前方案的致命缺陷：
1. **只是字符串过滤**：通过验证命令字符串来阻止危险操作
2. **可以被绕过**：运行脚本、解释器、编译代码都可以绕过
3. **没有真正的路径隔离**：`cd ..` 可以成功切换到上层目录

## 正确的解决方案

### 方案 1：Bubblewrap（推荐⭐⭐⭐⭐⭐）

**Bubblewrap** 是一个轻量级的 Linux 沙箱工具，专门设计用于在无特权用户环境中创建安全的容器化环境。

#### 核心优势
- ✅ **无需 root 权限**：使用 unprivileged user namespaces
- ✅ **完整的文件系统隔离**：Mount namespace 隔离
- ✅ **PID 隔离**：进程看不到外面的进程
- ✅ **网络隔离**：可选的网络 namespace
- ✅ **轻量级**：比 Docker 轻量得多
- ✅ **成熟稳定**：Flatpak、systemd 都在使用

#### 安装
```bash
# Ubuntu/Debian
sudo apt install bubblewrap

# CentOS/RHEL
sudo yum install bubblewrap

# Arch Linux
sudo pacman -S bubblewrap
```

#### 基本用法
```bash
bwrap \
  --unshare-all \
  --share-net \
  --dev /dev \
  --proc /proc \
  --dir /tmp \
  --ro-bind /etc /etc \
  --bind /workspaces/users/user123 /home \
  --chdir /home \
  --setenv HOME /home \
  --setenv USER user123 \
  /bin/bash
```

### 方案 2：unshare + chroot（需要 root）

使用 `unshare` 命令创建 namespaces，然后 chroot 到用户目录。

```bash
# 需要 root 权限或 CAP_SYS_ADMIN
unshare \
  --mount \
  --pid \
  --fork \
  --mount-proc \
  chroot /workspaces/users/user123 /bin/bash
```

### 方案 3：Docker/LXC（重量级但完整）

为每个用户创建一个轻量级容器。

```bash
docker run -it --rm \
  --user $(id -u):$(id -g) \
  --volume /workspaces/users/user123:/home \
  --workdir /home \
  --cap-drop=ALL \
  --read-only \
  alpine:latest /bin/sh
```

## 实现步骤

### 第一步：Bubblewrap 包装器

创建 `src/isolation/bubblewrap.ts`：

```typescript
import { spawn } from 'bun'

export interface BubblewrapConfig {
  userId: string
  userRoot: string
  cols: number
  rows: number
  enableNetwork?: boolean
  enableWriteAccess?: boolean
}

export function createBubblewrapShell(config: BubblewrapConfig) {
  const { userId, userRoot, cols, rows } = config
  
  // 构建 bubblewrap 参数
  const bwrapArgs = [
    '--unshare-all',           // 隔离所有 namespace
    config.enableNetwork ? '--share-net' : '--unshare-netns',
    
    // 文件系统隔离
    '--dev', '/dev',
    '--proc', '/proc',
    '--dir', '/tmp',
    '--ro-bind', '/etc', '/etc',
    '--ro-bind', '/usr', '/usr',
    '--ro-bind', '/bin', '/bin',
    
    // 绑定用户目录
    '--bind', userRoot, '/home',
    
    // 设置工作目录和环境
    '--chdir', '/home',
    '--setenv', 'HOME', '/home',
    '--setenv', 'USER', userId,
    '--setenv', 'TERM', 'xterm-256color',
    
    // 启动 shell
    '/bin/bash',
    '--norc',
    '--noprofile'
  ].filter(Boolean)
  
  // 使用 Bun.spawn 启动
  const proc = Bun.spawn(['bwrap', ...bwrapArgs], {
    terminal: {
      cols,
      rows,
      data: (term, data) => {
        // 处理输出
      }
    },
    stdio: ['pipe', 'pipe', 'pipe']
  })
  
  return proc
}
```

### 第二步：更新 PTY Manager

修改 `src/integration/ptyManager.ts`：

```typescript
import { createBubblewrapShell } from '../isolation/bubblewrap'

export class PTYSessionManager {
  createSession(options: PTYCreateOptions): PTYSession {
    const sessionId = uuidv4()
    const enableIsolation = options.enableIsolation ?? true
    
    if (enableIsolation && options.userId && options.userRoot) {
      // 使用 bubblewrap 创建隔离的 shell
      return this.createIsolatedSession(sessionId, options)
    } else {
      // 使用普通方式创建（不推荐）
      return this.createNormalSession(sessionId, options)
    }
  }
  
  private createIsolatedSession(
    sessionId: string,
    options: PTYCreateOptions
  ): PTYSession {
    const { userId, userRoot, cols, rows } = options
    
    // 使用 bubblewrap 创建隔离的 shell
    const proc = createBubblewrapShell({
      userId: userId!,
      userRoot: userRoot!,
      cols: cols || 120,
      rows: rows || 30,
      enableNetwork: true  // 可选是否启用网络
    })
    
    // 创建会话对象
    const session: PTYSession = {
      id: sessionId,
      userId,
      process: proc,
      cwd: userRoot!,
      isAlive: true,
      // ... 其他字段
    }
    
    this.sessions.set(sessionId, session)
    return session
  }
}
```

### 测试用例

## 部署考虑

### 系统要求

- Linux 内核 3.8+（支持 user namespaces）
- Bubblewrap 工具
- 启用 unprivileged user namespaces

### 配置

```bash
# /etc/sysctl.d/99-userns.conf
# 启用无特权 user namespaces
kernel.unprivileged_userns_clone=1
```

### 权限

Bubblewrap 的优势是**不需要 root 权限**，可以直接以普通用户身份运行。

## 回退方案

如果系统不支持 Bubblewrap，可以：

1. **使用 Docker**：重量级但完整
2. **使用 chroot**：需要 root，安全性较低
3. **纯软件验证**：当前的方案，作为最后手段
