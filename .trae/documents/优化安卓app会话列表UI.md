# 安卓app会话列表UI优化计划

## 目标
将当前会话列表UI（图1）优化为图2所示的样式，实现高度对齐的布局、改进的字体排版、优化的图标展示和预览文本显示。

## 当前问题
1. 使用 Box 布局配合绝对 padding 定位，布局不够灵活
2. 图标只是简单字母，不够直观
3. 时间显示位置和对齐不够精确
4. 预览文本生成逻辑简单，缺乏多样性
5. 列表项高度固定为 68dp，可能不够

## 优化方案

### 1. 布局重构
- 使用 `Row` + `Column` 替代当前的 Box 布局
- 采用标准列表项布局模式：左侧图标 → 中间内容（标题+预览）→ 右侧时间
- 确保三列内容水平垂直对齐
- 移除固定 padding 定位，改用 Flex 布局

### 2. 图标优化
- 使用 Material Icons 替代纯字母
- 根据会话标题类型显示不同图标：
  - Agent 相关 → `AutoAwesome` (AI图标)
  - 代码/开发 → `Code` (代码图标)
  - 数据/分析 → `BarChart` (图表图标)
  - 文件相关 → `Description` (文件图标)
  - 其他 → `SmartToy` 或 `Chat`
- 图标大小统一为 24dp
- 图标容器改为圆形，大小 40dp

### 3. 时间显示优化
- 时间右对齐，与列表项顶部对齐
- 字体大小保持 12sp
- 确保时间与图标顶部水平对齐

### 4. 文本排版优化
- 标题：15sp → 16sp，字体加粗
- 预览文本：13sp → 14sp，使用次要文字颜色
- 使用 `maxLines = 2` 让预览文本可显示两行
- 调整行间距，确保标题和预览文本之间有适当间距

### 5. 列表项间距
- 列表项高度从 68dp 调整为 72dp
- 添加列表项之间的分割线或间距
- 优化整体视觉层次

### 6. 预览文本生成改进
- 根据会话标题生成更丰富的预览文本
- 支持根据标题关键词动态生成预览
- 保持预览文本的自然语言风格

## 实施步骤

1. **修改 SessionItem 组件**
   - 重构为 Row + Column 布局
   - 实现三列对齐布局
   - 调整间距和对齐方式

2. **优化图标显示**
   - 添加 Material Icons 图标选择逻辑
   - 替换 getIconText 为 getIconResource 函数
   - 使用 Icon 组件替代 Text 显示图标

3. **调整时间显示**
   - 确保时间右对齐
   - 与标题顶部对齐
   - 优化时间格式显示

4. **优化文本样式**
   - 调整标题和预览文本的字号、颜色
   - 添加 maxLines 限制
   - 优化行间距

5. **改进预览文本生成**
   - 丰富预览文本内容
   - 根据标题类型生成不同预览
   - 保持预览文本的多样性

6. **测试验证**
   - 验证布局在不同屏幕尺寸下的表现
   - 检查暗色主题下的显示效果
   - 确认滑动删除功能正常工作

## 技术要点
- 使用 Compose 的 Row + Column 布局系统
- 使用 Material3 的 Icon 组件
- 保持现有主题系统兼容
- 确保滑动删除功能不受影响
- 保持响应式更新机制
