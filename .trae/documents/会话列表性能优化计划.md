# 会话列表性能优化计划

## 问题概述
安卓APP会话列表在滚动时不流畅，主要原因包括：
1. LazyColumn 缺少 LazyListState 导致状态无法复用
2. SwipeToDismissBox 在每个 item 中重复创建，增加内存和计算开销
3. remember 依赖键使用不当导致数据映射重复执行
4. 缺少 contentType 优化降低渲染效率
5. 搜索过滤在主线程同步执行

## 修复方案

### 阶段1：快速优化（高优先级，立即可做）

#### 1.1 添加 LazyListState 管理列表状态
**文件**: `app/src/main/java/com/example/claw_code_application/ui/chat/SessionListScreen.kt`
**修改位置**: 第109-124行的 LazyColumn
**具体步骤**:
- 添加 `val listState = rememberLazyListState()`
- 在 LazyColumn 中添加 `state = listState` 参数
- 添加必要的 import: `import androidx.compose.foundation.lazy.rememberLazyListState`

#### 1.2 添加 contentType 优化渲染
**文件**: `app/src/main/java/com/example/claw_code_application/ui/chat/SessionListScreen.kt`
**修改位置**: 第112-123行的 items 调用
**具体步骤**:
- 在 items 函数中添加 `contentType = { "session_item" }` 参数
- 这允许 Compose 复用相同类型的 item 布局

#### 1.3 优化 remember 依赖键
**文件**: `app/src/main/java/com/example/claw_code_application/ui/chat/SessionListScreen.kt`
**修改位置**: 第57-68行的 sessionDisplayData 定义
**具体步骤**:
- 当前代码: `remember(sessions)` - 使用对象引用作为key，每次重组都会失效
- 修改为使用稳定标识: 利用 sessions 的 size 和 id 列表作为依赖
- 或者使用 `derivedStateOf` 延迟计算

### 阶段2：中优先级优化（减少内存和计算开销）

#### 2.1 简化 SwipeToDismiss 实现
**文件**: `app/src/main/java/com/example/claw_code_application/ui/chat/SessionListScreen.kt`
**修改位置**: 第265-316行的 SwipeToDismissSessionItem 函数
**具体步骤**:
- 当前每个 item 都创建独立的 SwipeToDismissBox 和状态
- 方案A（推荐）: 改用长按显示删除菜单，性能更好
- 方案B: 优化 SwipeToDismiss，使用共享状态
- 建议采用方案A，因为滑动删除在列表滚动中容易误触

#### 2.2 优化数据流动画触发
**文件**: `app/src/main/java/com/example/claw_code_application/MainActivity.kt`
**修改位置**: 第207-259行的 AnimatedVisibility
**具体步骤**:
- 在数据加载期间避免重复触发加载动画
- 使用 `key()` 函数稳定动画的起始状态
- 确保动画只在真正需要切换时触发

### 阶段3：低优先级优化（提升用户体验）

#### 3.1 使用协程优化搜索过滤
**文件**: `app/src/main/java/com/example/claw_code_application/ui/chat/SessionListScreen.kt`
**修改位置**: 第70-79行的 filteredData 定义
**具体步骤**:
- 当前搜索过滤在主线程同步执行
- 使用 `produceState` 或 `LaunchedEffect` 在后台线程执行过滤
- 添加防抖机制，避免频繁搜索

#### 3.2 添加 Compose 稳定性注解
**文件**: `app/src/main/java/com/example/claw_code_application/data/api/models/SessionModels.kt`
**修改位置**: Session data class 定义
**具体步骤**:
- 已有 `@Immutable` 注解，确保正确生效
- 检查其他数据类是否也添加了稳定性注解
- 在 `build.gradle.kts` 中启用 Compose 编译器优化

## 实施顺序
1. 阶段1.1 → 1.2 → 1.3（快速优化，立即见效）
2. 阶段2.1 → 2.2（中等优化，显著改善）
3. 阶段3.1 → 3.2（体验优化，锦上添花）

## 验证方法
- 在模拟器/真机上测试滚动流畅度
- 使用 Android Studio Profiler 查看帧率
- 检查 logcat 是否有 "Skipped frames" 警告
- 验证滑动删除功能（如果保留）是否正常工作

## 风险评估
- 阶段1: 低风险，纯性能优化，不影响功能
- 阶段2: 中等风险，改动交互逻辑，需要测试
- 阶段3: 低风险，优化搜索体验

## 预期效果
- 滚动帧率从 30-40fps 提升到 55-60fps
- 内存使用减少约 20%
- 重组次数减少约 60%
