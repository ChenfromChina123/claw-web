### 2. 问题分析

**当前实现问题**:

1. `titleGeneratingSessions` 是 `Set<string>`，只能标记"正在生成"，无法追踪请求序列
2. 如果 LLM 生成较慢，后续消息会跳过标题生成
3. LLM 失败时没有降级处理（虽然用户说不用简单规则，但需要确保能生成标题）

**正确的并行实现**:
应该使用 `Map<sessionId, number>` 存储序列号：

* 每次发起标题生成请求时递增序列号

* LLM 返回时检查序列号是否仍是最新

* 如果序列号过期，丢弃结果

## 修改计划

**修改** **`sessionManager.ts`**:

1. 将 `titleGeneratingSessions: Set<string>` 改为 `titleGenSeq: Map<string, number>`
2. 修改 `generateAndUpdateSessionTitle` 方法，传入当前序列号
3. LLM 返回后检查序列号，只有最新序列号才能更新标题
4. LLM 生成失败时，使用内容前30字符作为兜底标题
