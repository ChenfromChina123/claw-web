### 二、角色职责划分

#### 2.1 Master 职责 (独占)

| 模块 | 功能 | 位置 |
|------|------|------|
| **HTTP/WebSocket 服务** | 接收客户端请求，提供 API 端点 | `server/src/server/httpServer.ts` |
| **认证与授权** | 用户登录、JWT 验证、权限管理 | `server/src/services/authService.ts` |
| **数据库访问** | MySQL 连接池、CRUD 操作 | `server/src/db/mysql.ts` |
| **LLM 服务** | 调用 Claude/GPT 等大模型 | `server/src/services/llmService.ts` |
| **会话管理** | 会话持久化、消息历史 | `server/src/services/sessionManager.ts` |
| **容器编排** | Docker 容器生命周期管理、热池维护 | `server/src/orchestrator/containerOrchestrator.ts` |
| **工作空间管理** | 工作区创建、快照管理 | `server/src/services/workspaceManager.ts` |
| **技能系统** | 内置技能 + 用户技能管理 | `server/src/skills/` |
| **MCP 网关** | Model Context Protocol 集成 | `server/src/services/mcp/McpGateway.ts` |
| **监控告警** | 性能指标收集、日志管理 | `server/src/routes/monitoring.routes.ts` |
| **插件系统** | 插件加载与管理 | `server/src/integrations/plugins/` |

#### 2.2 Worker 职责 (纯沙箱)

| 模块 | 功能 | 位置 |
|------|------|------|
| **Internal API** | 处理 Master 转发的请求 | `server/src/worker/server/index.ts` |
| **PTY 管理** | 终端会话创建与 I/O | `server/src/worker/terminal/ptyManager.ts` |
| **Sandbox 执行** | 命令执行、文件操作（限制在 /workspace） | `server/src/worker/sandbox/index.ts` |
| **健康检查** | 容器健康状态上报 | `/internal/health` |

**Worker 特点：**
- **无数据库连接** - 所有 DB 操作由 Master 处理
- **无 LLM 调用** - LLM 请求通过 Master 转发
- **受限文件系统** - 仅能访问 `/workspace` 目录
- **Token 验证** - 仅信任带 `X-Master-Token` 的请求

### 五、通信机制

#### 5.1 Master → Worker 通信

**Internal API 端点 (Worker 监听 4000 端口)：**

```
POST /internal/exec          - 执行命令
POST /internal/pty/create    - 创建 PTY 会话
POST /internal/pty/write     - 写入 PTY
POST /internal/pty/resize    - 调整 PTY 大小
POST /internal/pty/destroy   - 销毁 PTY
POST /internal/file/read      - 读取文件
POST /internal/file/write     - 写入文件
POST /internal/file/list      - 列出目录
GET  /internal/health         - 健康检查
```

**请求头部：**
- `X-Master-Token` - Master 内部通信令牌
- `X-User-Id` - 用户 ID
- `X-User-Admin` - 是否管理员

### 六、安全隔离

#### 6.1 网络隔离

- **Worker 网络**：`claw-web_worker-network`
- **MySQL 仅在 Master 网络可见**，Worker 无法直接访问数据库

#### 6.2 文件系统隔离

- Worker 只能访问 `/workspace` 目录
- 路径安全检查：`isPathSafe()` 函数防止目录遍历

#### 6.3 资源限制

Docker 容器资源限制通过 `hardwareResourceConfig` 管理：

| 用户等级 | 内存限制 | 存储配额 | PTY 进程数 | 文件数 |
|---------|---------|---------|-----------|-------|
| free    | 512MB   | 500MB   | 2         | 500   |
| pro     | 1GB     | 2GB     | 5         | 2000  |
| enterprise | 2GB  | 10GB    | 10        | 10000 |

#### 6.4 Docker 安全加固

```bash
--read-only                    # 根文件系统只读
--security-opt=no-new-privileges
--cap-drop=ALL                 # 移除所有 Linux 能力
--pids-limit 100               # 限制进程数
--ulimit nproc=100:100         # 限制用户进程数
--ulimit nofile=1024:1024      # 限制文件描述符
```

### 九、架构优势

| 优势 | 说明 |
|------|------|
| **水平扩展** | Worker 容器可按需扩缩，应对流量高峰 |
| **资源隔离** | 每个用户独享一个容器，环境互不干扰 |
| **安全加固** | Worker 无 DB 访问权限，防止数据泄露 |
| **热启动** | 热池预启动容器，用户请求零等待 |
| **角色分离** | Master 处理复杂业务逻辑，Worker 专注执行 |
