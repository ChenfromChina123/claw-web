# 容器内项目部署与外部访问功能实施计划

## 一、需求分析

### 1.1 核心需求

* **容器内项目部署**：用户可以在容器内直接部署和运行项目

* **持久化运行**：项目进程能够持久化运行，容器重启后自动恢复

* **外部访问**：部署的项目能够被外部网络访问

### 1.2 技术挑战

1. **进程管理**：容器内如何管理和守护进程
2. **端口管理**：动态分配和管理多个项目的端口
3. **网络访问**：如何安全地暴露服务到外部网络
4. **资源隔离**：多用户环境下的资源限制和隔离
5. **安全性**：防止恶意代码执行和未授权访问

## 二、技术方案调研

### 2.0 Manus 技术架构参考

**Manus 核心技术理念**：
Manus 是全球首款通用型 AI Agent 产品，采用了 Multiple Agent 架构，其核心技术特点包括：

1. **分布式多智能体架构（MAS）**：

   * **规划代理（Planning Agent）**：任务拆解至 0.1 秒级决策

   * **执行代理（Execution Agent）**：多工具协同调用

   * **验证代理（Validation Agent）**：质量评估、交付成果

2. **沙箱虚拟机隔离**：

   * 完全运行在独立虚拟机中

   * 确保安全性与并行效率

   * Docker 容器隔离机制

3. **Deep Agent 范式**：

   * 完全自主地完成端到端任务

   * 不需要人工介入中间步骤

   * 输出可交付的最终成果而非建议

4. **技术优势**：

   * GAIA 测试 SOTA（比 OpenAI 产品成功率高 25%，执行速度快 30%）

**应用到本项目的启示**：

### 2.1 进程持久化方案对比

| 方案                   | 优点                  | 缺点          | 适用场景        |
| -------------------- | ------------------- | ----------- | ----------- |
| **PM2**              | 自动重启、日志管理、监控面板、负载均衡 | 需要Node.js环境 | Node.js应用   |
| **Supervisor**       | 轻量级、配置简单、支持多种进程     | 功能相对简单      | Python/通用进程 |
| **Systemd**          | 系统级、功能强大、自动依赖管理     | 容器内权限限制     | 系统服务        |
| **Docker-in-Docker** | 完全隔离、资源限制好          | 安全风险、复杂度高   | 需要完全隔离的场景   |
| **自定义进程管理器**         | 可定制、与现有系统集成         | 开发成本高       | 特殊需求        |

**推荐方案**：使用 **PM2**（Node.js项目）+ **Supervisor**（其他类型项目）

### 2.2 外部访问方案对比

| 方案                    | 成本  | 安全性 | 易用性 | 适用场景         |
| --------------------- | --- | --- | --- | ------------ |
| **端口映射 + 反向代理**       | 低   | 中   | 中   | 有公网IP的服务器    |
| **Cloudflare Tunnel** | 免费  | 高   | 高   | 无公网IP、需要安全访问 |
| **内网穿透（cpolar/frp）**  | 低/中 | 中   | 中   | 无公网IP、临时访问   |
| **VPN（Tailscale）**    | 免费  | 高   | 高   | 团队内部访问       |
| **混合方案**              | 低   | 高   | 中   | 多种访问需求       |

**推荐方案**：

1. **生产环境**：反向代理（Nginx） + SSL + Cloudflare Tunnel
2. **开发环境**：内网穿透（cpolar/frp）
3. **团队协作**：VPN（Tailscale）

#### 任务 2：在 Worker 容器内集成进程管理器

**设计理念**：

* **容器内进程管理**：在已运行的 Worker 容器内管理多个项目进程

* **端口动态分配**：在容器内为不同项目分配不同端口

* **资源隔离**：通过进程管理器实现项目间的资源隔离

* **复用现有基础设施**：利用 Worker 容器的存储和网络配置

**文件位置**：`server/src/integration/processManager.ts`

**功能**：

1. PM2 集成（Node.js 项目）
2. Supervisor 集成（Python/通用项目）
3. 进程状态监控
4. 自动重启机制
5. 日志收集和轮转

**实现方案**：

```typescript
// Worker 容器内的进程管理器
class WorkerProcessManager {
  /**
   * 在 Worker 容器内启动项目进程
   * 通过 Docker exec 在容器内执行命令
   */
  async startProjectInWorker(
    workerContainerId: string,
    projectId: string,
    config: ProjectConfig
  ): Promise<{ port: number; processId: string }> {
    // 1. 分配容器内端口
    const internalPort = await this.allocateInternalPort(workerContainerId)
    
    // 2. 准备项目目录（在 Worker 容器内）
    const projectPath = `/app/workspaces/users/${config.userId}/projects/${projectId}`
    await this.execInContainer(workerContainerId, `mkdir -p ${projectPath}`)
    
    // 3. 根据项目类型选择进程管理器
    if (config.type === 'nodejs') {
      return await this.startWithPM2InContainer(
        workerContainerId,
        projectId,
        projectPath,
        internalPort,
        config
      )
    } else {
      return await this.startWithSupervisorInContainer(
        workerContainerId,
        projectId,
        projectPath,
        internalPort,
        config
      )
    }
  }

  /**
   * 在容器内使用 PM2 启动 Node.js 项目
   */
  private async startWithPM2InContainer(
    containerId: string,
    projectId: string,
    projectPath: string,
    port: number,
    config: ProjectConfig
  ): Promise<{ port: number; processId: string }> {
    // 生成 PM2 配置文件
    const pm2Config = {
      name: projectId,
      script: config.startCommand,
      cwd: projectPath,
      instances: 1,
      autorestart: true,
      watch: false,
      max_memory_restart: config.memoryLimit || '256M',
      env: {
        PORT: port,
        NODE_ENV: 'production',
        ...config.envVars
      },
      error_file: `/app/logs/${projectId}-error.log`,
      out_file: `/app/logs/${projectId}-out.log`
    }

    // 将配置写入容器
    await this.execInContainer(
      containerId,
      `echo '${JSON.stringify(pm2Config)}' > ${projectPath}/ecosystem.config.js`
    )

    // 使用 PM2 启动
    const { stdout } = await this.execInContainer(
      containerId,
      `cd ${projectPath} && pm2 start ecosystem.config.js`
    )

    return { port, processId: projectId }
  }

  /**
   * 在容器内使用 Supervisor 启动项目
   */
  private async startWithSupervisorInContainer(
    containerId: string,
    projectId: string,
    projectPath: string,
    port: number,
    config: ProjectConfig
  ): Promise<{ port: number; processId: string }> {
    // 生成 Supervisor 配置
    const supervisorConfig = `
[program:${projectId}]
command=${config.startCommand}
directory=${projectPath}
autostart=true
autorestart=true
stderr_logfile=/app/logs/${projectId}-error.log
stdout_logfile=/app/logs/${projectId}-out.log
environment=PORT="${port}",${Object.entries(config.envVars).map(([k, v]) => `${k}="${v}"`).join(',')}
user=bun
`

    // 写入配置文件
    await this.execInContainer(
      containerId,
      `echo '${supervisorConfig}' > /etc/supervisor/conf.d/${projectId}.conf`
    )

    // 重新加载 Supervisor
    await this.execInContainer(containerId, 'supervisorctl reread')
    await this.execInContainer(containerId, 'supervisorctl update')

    return { port, processId: projectId }
  }

  /**
   * 在容器内执行命令
   */
  private async execInContainer(
    containerId: string,
    command: string
  ): Promise<{ stdout: string; stderr: string }> {
    const { exec } = require('child_process')
    const { promisify } = require('util')
    const execAsync = promisify(exec)

    return await execAsync(`docker exec ${containerId} ${command}`)
  }

  /**
   * 分配容器内端口（避免冲突）
   */
  private async allocateInternalPort(containerId: string): Promise<number> {
    // 查询容器内已使用的端口
    const { stdout } = await this.execInContainer(
      containerId,
      'netstat -tuln | grep LISTEN | awk \'{print $4}\' | cut -d: -f2'
    )
    
    const usedPorts = stdout.split('\n').map(p => parseInt(p)).filter(p => !isNaN(p))
    
    // 从 3100 开始分配（容器内部端口范围）
    for (let port = 3100; port < 3200; port++) {
      if (!usedPorts.includes(port)) {
        return port
      }
    }
    
    throw new Error('容器内端口已用完')
  }

  /**
   * 停止容器内的项目进程
   */
  async stopProjectInWorker(
    workerContainerId: string,
    projectId: string,
    processManager: 'pm2' | 'supervisor'
  ): Promise<void> {
    if (processManager === 'pm2') {
      await this.execInContainer(workerContainerId, `pm2 stop ${projectId}`)
      await this.execInContainer(workerContainerId, `pm2 delete ${projectId}`)
    } else {
      await this.execInContainer(workerContainerId, `supervisorctl stop ${projectId}`)
      await this.execInContainer(workerContainerId, `rm /etc/supervisor/conf.d/${projectId}.conf`)
      await this.execInContainer(workerContainerId, 'supervisorctl update')
    }
  }

  /**
   * 获取容器内项目状态
   */
  async getProjectStatusInWorker(
    workerContainerId: string,
    projectId: string,
    processManager: 'pm2' | 'supervisor'
  ): Promise<ProcessStatus> {
    if (processManager === 'pm2') {
      const { stdout } = await this.execInContainer(
        workerContainerId,
        `pm2 jlist`
      )
      const processes = JSON.parse(stdout)
      const process = processes.find((p: any) => p.name === projectId)
      
      return {
        running: process?.pm2_env?.status === 'online',
        pid: process?.pid,
        memory: process?.monit?.memory,
        cpu: process?.monit?.cpu,
        uptime: process?.pm2_env?.pm_uptime
      }
    } else {
      const { stdout } = await this.execInContainer(
        workerContainerId,
        `supervisorctl status ${projectId}`
      )
      
      const isRunning = stdout.includes('RUNNING')
      return {
        running: isRunning,
        status: stdout.trim()
      }
    }
  }

  /**
   * 获取容器内项目日志
   */
  async getProjectLogsInWorker(
    workerContainerId: string,
    projectId: string,
    lines: number = 100
  ): Promise<{ stdout: string; stderr: string }> {
    const { stdout } = await this.execInContainer(
      workerContainerId,
      `tail -n ${lines} /app/logs/${projectId}-out.log`
    )
    
    const { stdout: stderr } = await this.execInContainer(
      workerContainerId,
      `tail -n ${lines} /app/logs/${projectId}-error.log`
    )
    
    return { stdout, stderr }
  }
}
```

## 四、技术选型建议

### 4.1 进程管理器选择

**推荐方案**：

1. **Node.js 项目**：PM2

   * 优势：原生支持、功能完善、监控面板

   * 配置示例：

     ```javascript
     {
       name: 'my-app',
       script: 'app.js',
       instances: 1,
       autorestart: true,
       watch: false,
       max_memory_restart: '1G',
       env: {
         NODE_ENV: 'production',
         PORT: 3101
       }
     }
     ```

2. **Python 项目**：Supervisor

   * 优势：轻量级、配置简单、支持多种进程

   * 配置示例：

     ```ini
     [program:my-python-app]
     command=python app.py
     directory=/app/projects/my-python-app
     autostart=true
     autorestart=true
     stderr_logfile=/var/log/my-python-app.err.log
     stdout_logfile=/var/log/my-python-app.out.log
     environment=PORT="3102"
     ```

3. **Docker 项目**：Docker-in-Docker（谨慎使用）

   * 优势：完全隔离、资源限制好

   * 注意：安全风险较高，需要额外配置

### 4.2 外部访问方案选择

**推荐方案**（按优先级）：

1. **Cloudflare Tunnel**（强烈推荐）

   * 优势：免费、安全、无需公网 IP、自动 SSL

   * 适用场景：生产环境、需要安全访问

   * 实现方式：

     ```bash
     # 安装 cloudflared
     docker run -d --name cloudflared \
       -v ~/.cloudflared:/etc/cloudflared \
       cloudflare/cloudflared:latest \
       tunnel --no-autoupdate run --token <token>
     ```

2. **反向代理 + SSL**

   * 优势：可控性强、性能好

   * 适用场景：有公网 IP 的服务器

   * 实现方式：

     ```nginx
     server {
         listen 443 ssl http2;
         server_name myapp.example.com;
         
         ssl_certificate /etc/letsencrypt/live/myapp.example.com/fullchain.pem;
         ssl_certificate_key /etc/letsencrypt/live/myapp.example.com/privkey.pem;
         
         location / {
             proxy_pass http://localhost:3101;
             proxy_set_header Host $host;
             proxy_set_header X-Real-IP $remote_addr;
         }
     }
     ```

3. **内网穿透（cpolar/frp）**

   * 优势：无需公网 IP、配置简单

   * 适用场景：开发测试、临时访问

   * 实现方式：

     ```bash
     # cpolar
     docker run -d --name cpolar \
       -p 9200:9200 \
       -v /var/lib/cpolar:/var/lib/cpolar \
       cpolar/cpolar:latest

     # 创建隧道
     curl http://localhost:9200/api/v1/tunnels \
       -H "Authorization: Bearer <token>" \
       -d '{"addr":"3101","proto":"http"}'
     ```

4. **VPN（Tailscale）**

   * 优势：安全性最高、团队协作友好

   * 适用场景：团队内部访问

   * 实现方式：

     ```bash
     docker run -d --name tailscaled \
       -v /var/lib/tailscale:/var/lib/tailscale \
       -v /dev/net/tun:/dev/net/tun \
       --cap-add=NET_ADMIN \
       tailscale/tailscale:latest
     ```

### 4.3 持久化存储方案

**推荐方案**：

1. **Docker Volumes**（推荐）

   * 优势：性能好、管理方便

   * 配置示例：

     ```yaml
     volumes:
       - project_data:/app/projects
       - project_logs:/var/log/projects
       - project_config:/etc/projects
     ```

2. **Bind Mounts**

   * 优势：直接访问宿主机文件

   * 配置示例：

     ```yaml
     volumes:
       - ./data/projects:/app/projects
       - ./data/logs:/var/log/projects
     ```

## 十、总结

本方案通过以下技术组合实现用户需求：

### 10.1 核心技术方案

1. **项目部署**：

   * **复用现有架构**：基于项目已有的 Master-Worker 容器架构

   * **Worker 内部署**：在已分配的 Worker 容器内运行多个项目

   * **进程管理**：PM2（Node.js）+ Supervisor（Python/通用）

   * **Manus 智能体**：规划-执行-验证三阶段部署流程

2. **持久化运行**：

   * **自动重启**：PM2/Supervisor 提供进程守护

   * **健康检查**：ContainerOrchestrator 监控容器状态

   * **监控告警**：实时监控项目运行状态

   * **日志管理**：集中收集和轮转日志

3. **外部访问**：

   * **反向代理**：Nginx 动态路由到不同 Worker

   * **SSL 证书**：Let's Encrypt 自动化证书管理

   * **域名管理**：支持自定义域名和子域名

### 10.2 架构优势

**基于现有架构的独特优势**：

2. **快速部署**：

   * 项目在已运行的容器内启动，秒级响应

   * 基于现有的用户-容器映射机制

   * 通过 ContainerOrchestrator 统一管理所有容器
