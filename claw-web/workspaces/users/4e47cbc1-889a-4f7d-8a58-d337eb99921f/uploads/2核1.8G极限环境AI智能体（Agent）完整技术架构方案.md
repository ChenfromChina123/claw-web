## 二、核心隐形风险与根治方案

AI Agent架构四大易忽略崩溃点，针对性最佳实践：

### 1. 绝对路径与相对路径认知断层

- 问题：AI认知路径与沙箱物理路径不一致，生成错误代码、触发危险操作

- 方案：

    1. System Prompt强制AI仅用**相对路径（./）**

    2. 后端搭建**路径转换层**，拦截路径并完成沙箱→物理路径映射校验

### 2. 终端输出无效信息过载

- 问题：ANSI颜色码、进度条字符耗尽Token，拖慢低内存后端处理

- 方案：

    1. 用`strip-ansi`过滤特殊控制字符

    2. 命令强制加静默参数（`npm install --no-progress`），减少冗余输出

### 3. 文件系统快照与AI记忆同步失调

- 问题：AI认知与实际文件状态脱节，重复修改陷入死循环

- 方案：

    1. 文件操作后返回**强状态反馈**（更新结果、大小、校验和）

    2. 关键操作前强制AI重新读取目录/文件，同步最新状态

### 4. 孤儿进程与内存泄露

- 问题：后台僵尸进程耗尽1.8G内存，服务直接崩溃

- 方案：

    1. 用`tree-kill`清理进程树，主进程终止时同步杀死所有子进程

    2. 通过`ulimit`限制沙箱最大进程数，源头管控

## 四、核心执行底座：轻量沙箱隔离（bwrap + cgroups）

### 方案选型

摒弃高开销Docker、低安全chroot，选用**Bubblewrap（bwrap）+ Linux cgroups**，安全性拉满、比Docker省100-200MB内存，毫秒级启动。

### 实施流程

1. **空间分配**：为每个任务创建**UUID唯一目录**，所有操作限定于此，任务结束自动清理

2. **最小化环境**：提前准备只读基础模板（/opt/ai-template），仅内置ls、python、node等必备工具

3. **文件系统隔离**

    - 只读挂载：`--ro-bind`绑定/bin、/lib、/usr，防止篡改系统文件

    - 读写挂载：`--bind`仅开放UUID任务目录，限制操作范围

    - 私有空间：独立/tmp目录，彻底隔绝内外环境

4. **资源配额限制**

    - 创建`ai_agents`控制组，单沙箱内存上限512M、CPU使用率50%

    - 沙箱PID绑定控制组，资源限制精准生效

5. **网络隔离**

    - 完全隔离：`--unshare-net`关闭外网权限

    - 按需放行：用slirp4netns提供受限网络接入

### 内存优化细节

- Bind Mount指针挂载，不复制文件，零额外内存开销

- 配置vm.swappiness=10，优先物理内存，Swap兜底瞬时峰值

- 共享Python/Node运行时，多沙箱共用，减少重复加载

## 七、核心模块精细化实现

### 1. 终端运行管控（低内存适配）

- **长输出处理**：首尾截断+中间省略，仅保留前50行+后100行，提供分页读取工具

- **长耗时任务**：异步Job模式，返回job_id，SSE推送关键状态，不传输全量日志

- **进程管控**：维护`Map<job_id, ChildProcess>`，超时/出错时SIGINT/SIGKILL强制终止

- **无长连接**：不维持PTY长连接，依托固定目录持久化运行，节省内存

### 2. 文件读取管控（防爆Token/内存）

- **元数据先行**：先查文件大小、格式、行数，避免无效读取

- **分片读取**：`read_file_lines`按行区间读取，单次加载少量内容

- **二进制文件**：图片用多模态模型解析，编译类文件直接拒绝读取

### 3. 项目检索（分层轻量方案）

1.8G内存不支持向量库，采用**传统检索+轻量语义过滤**：

- 文件搜索：Linux `find`快速定位

- 关键词搜索：ripgrep(rg)极速检索，带行号+上下文

- 语义搜索：Orama轻量库，仅对文件摘要做内存匹配

- 结果限流：匹配超20项仅返回文件名，引导AI单文件精读

### 4. 文件编辑（精准高效，省Token）

摒弃全量覆盖，采用**Lazy Surgical Edit（精确行号+短锚校验）**：

1. AI指定编辑行号区间+首行前10字符短锚点

2. 后端校验锚点，行号偏移则上下5-10行模糊匹配

3. 仅修改指定行，后端自动用Prettier/ESLint格式化

- 优势：比全量替换省80%Token，适配低内存环境

### 5. 实时状态同步

- **流式日志**：WebSocket/SSE推送AI思考、终端命令、执行结果

- **文件树同步**：磁盘操作后自动扫描目录，前端实时渲染文件结构

## 八、安全兜底：Git影子仓库机制

### 核心方案

原生Git + Bun.$调用，内存开销极低，为AI提供**后悔药**：

1. **沙箱初始化**：任务启动时`git init`，配置局部身份，完成初始提交

2. **操作前快照**：危险操作前自动`git add . && git commit`，留存状态

3. **撤销工具**：`git reset --hard HEAD^`快速回滚误操作

4. **差异查看**：`git diff`返回精简变更，节省Token

### 避坑细节

- 自动生成.gitignore，屏蔽node_modules、dist等大文件

- 非交互式执行，避免进程挂起

- 仅危险操作前提交，减少磁盘IO

## 九、性能优化：前缀缓存（Prompt Caching）

### 核心原理

LLM复用固定前缀的KV Cache，降低延迟、节省成本，1.8G服务器必备。

### 缓存友好型Prompt结构（由静到动）

1. System Prompt（恒定）

2. 工具定义（恒定）

3. 项目元数据（半静态）

4. 状态摘要（低频变动）

5. 对话历史（高频变动）

6. 用户最新指令（实时）

### 关键规则

- 禁止在System Prompt加动态时间戳，避免缓存失效

- 块对齐：按1024 Token填充缓存块，提升命中率

- 仅文件变动时更新文件树，减少前缀刷新

- **前端优化**：虚拟列表渲染、信号量限流，单任务串行执行

### 2. 硬性资源限制

- **串行执行**：同一时间仅处理1个活跃任务，杜绝资源争抢

- **强制超时**：单命令60秒超时熔断，防止死循环

- **内存监控**：cgroups超标自动终止，返回内存溢出错误

- **工具限流**：检索工具限制深度/大小，防止CPU/IO飙升

- `bun build --compile`编译为二进制包，省80MB内存

- 定时`Bun.gc(true)`强制垃圾回收，清理内存碎片
