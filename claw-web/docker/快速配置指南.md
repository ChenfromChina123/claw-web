# Docker 国内镜像快速配置指南

## 🚀 快速解决方案

### 方法一：使用自动化脚本（推荐）

1. **以管理员身份打开 PowerShell**
2. **运行配置脚本**：
   ```powershell
   cd d:\Users\Administrator\AistudyProject\HAHA\claw-web\docker
   .\configure-docker-mirror.ps1
   ```
3. **重启 Docker Desktop**
4. **重新构建项目**：
   ```bash
   docker compose -f docker-compose.cn-fast.yml up -d --build
   ```

### 方法二：手动配置 Docker Desktop

1. 打开 **Docker Desktop**
2. 点击右上角 **Settings** ⚙️
3. 选择左侧 **Docker Engine**
4. 在 JSON 配置中添加或修改：

```json
{
  "registry-mirrors": [
    "https://docker.mirrors.ustc.edu.cn",
    "https://registry.docker-cn.com",
    "https://hub-mirror.c.163.com"
  ],
  "builder": {
    "gc": {
      "defaultKeepStorage": "20GB",
      "enabled": true
    }
  },
  "experimental": false
}
```

5. 点击 **Apply & Restart**

### 方法三：使用国内镜像版本

项目已提供使用国内镜像的配置文件：

```bash
# 使用国内镜像加速版本
docker compose -f docker-compose.cn-fast.yml up -d --build

# 或使用原来的国内版本
docker compose -f docker-compose.cn.yml up -d --build
```

## 📋 详细说明

### 为什么需要镜像加速？

Docker Hub 服务器位于海外，国内访问速度慢且不稳定。配置镜像加速器后，Docker 会从国内的镜像站点拉取镜像，速度可提升 10 倍以上。

### 构建时间对比

| 配置方式 | 基础镜像下载 | 完整构建 | 说明 |
|---------|------------|---------|------|
| 默认（无加速） | 20-40 分钟 | 30-60 分钟 | 受网络影响大 |
| 配置镜像加速 | 2-5 分钟 | 5-15 分钟 | 速度提升 4-6 倍 |
| 手动预拉取 | 1-3 分钟 | 3-10 分钟 | 最稳定 |

### 项目中的镜像优化

项目已经在以下文件中配置了国内镜像源：

1. **Dockerfile** 中已配置：
   - `server/Dockerfile`: 使用阿里云 apt 源 + npm 镜像
   - `web/Dockerfile`: 使用阿里云 apk 源 + npm 镜像
   - `Dockerfile`: 使用 npm 镜像

2. **docker-compose** 文件：
   - `docker-compose.cn.yml`: MySQL 使用阿里云镜像
   - `docker-compose.cn-fast.yml`: 完整优化版本

## 🔧 故障排查

### 问题 1：构建卡在 "pulling image"

**症状**：
```
=> [internal] load metadata for docker.io/oven/bun:1
... (长时间无响应)
```

**解决方案**：
```bash
# 1. 手动拉取镜像
docker pull ccr.ccs.tencentyun.com/oven/bun:1
docker tag ccr.ccs.tencentyun.com/oven/bun:1 oven/bun:1

# 2. 重新构建
docker compose -f docker-compose.cn-fast.yml build
```

### 问题 2：SSL 证书错误

**症状**：
```
x509: certificate is not valid for any names
```

**解决方案**：
在 Dockerfile 中使用 HTTP 协议而非 HTTPS：
```dockerfile
# 修改前
RUN apk add --update https://mirrors.aliyun.com/...

# 修改后
RUN apk add --update http://mirrors.aliyun.com/...
```

### 问题 3：镜像标签不匹配

**症状**：
```
manifest for oven/bun:1 not found
```

**解决方案**：
```bash
# 从国内源拉取并重命名
docker pull registry.cn-hangzhou.aliyuncs.com/library/node:20-alpine
docker tag registry.cn-hangzhou.aliyuncs.com/library/node:20-alpine node:20-alpine
```

## 🎯 最佳实践

### 1. 预拉取常用镜像

```bash
# 创建脚本 pull-images.sh
docker pull registry.cn-hangzhou.aliyuncs.com/library/node:20-alpine
docker pull registry.cn-hangzhou.aliyuncs.com/library/nginx:1.25-alpine
docker pull ccr.ccs.tencentyun.com/oven/bun:1
docker pull registry.cn-hangzhou.aliyuncs.com/library/mysql:8.0
```

### 2. 定期清理缓存

```bash
# 清理未使用的镜像
docker image prune -a

# 清理构建缓存
docker builder prune -a

# 清理停止的容器
docker container prune
```

### 3. 使用构建缓存

```bash
# 启用 BuildKit
$env:DOCKER_BUILDKIT=1

# 使用缓存构建
docker compose -f docker-compose.cn-fast.yml build --progress=plain

# 无缓存重新构建
docker compose -f docker-compose.cn-fast.yml build --no-cache
```

## 📞 获取帮助

如果遇到问题：

1. 查看详细构建日志：
   ```bash
   docker compose -f docker-compose.cn-fast.yml build --progress=plain
   ```

2. 检查 Docker 配置：
   ```bash
   docker info | Select-String "Registry Mirrors"
   ```

3. 验证镜像源是否可用：
   ```bash
   curl -I https://docker.mirrors.ustc.edu.cn
   ```
