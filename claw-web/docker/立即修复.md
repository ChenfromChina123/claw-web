# ⚡ 立即修复 - Docker 构建失败

## 问题
```
✘ Image registry.cn-hangzhou.aliyuncs.com/library/mysql:8.0 Error
pull access denied, repository does not exist or may require 'docker login'
```

## ✅ 3 步快速修复

### 步骤 1：配置 Docker 镜像加速器

**打开 Docker Desktop** → **Settings** → **Docker Engine**

**粘贴以下配置**：

```json
{
  "registry-mirrors": [
    "https://docker.m.daocloud.io",
    "https://docker.1panel.live"
  ],
  "builder": {
    "gc": {
      "defaultKeepStorage": "20GB",
      "enabled": true
    }
  },
  "experimental": false
}
```

**点击** **Apply & Restart**

### 步骤 2：停止当前构建

```bash
docker compose -f docker-compose.cn-fast.yml down
```

### 步骤 3：重新构建

```bash
docker compose -f docker-compose.cn-fast.yml up -d --build
```

## 🆘 仍然失败？

### 临时方案：使用原始配置文件

```bash
# 使用原始的 docker-compose.cn.yml
docker compose -f docker-compose.cn.yml up -d --build
```

### 手动拉取镜像

```bash

docker pull mysql:8.0

# 然后重新构建

```

