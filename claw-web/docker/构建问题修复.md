# ⚡ Docker 构建问题紧急修复

## ❌ 问题说明

阿里云镜像仓库 `registry.cn-hangzhou.aliyuncs.com/library/mysql:8.0` 不存在或需要登录，导致构建失败。

## ✅ 解决方案

### 方案一：配置 Docker 镜像加速器（推荐）

**这是最可靠的解决方案**，配置后所有 Docker 镜像都会加速。

#### Windows Docker Desktop 配置步骤：

1. 打开 **Docker Desktop**
2. 点击右上角 **Settings** ⚙️
3. 选择左侧 **Docker Engine**
4. 将以下 JSON 粘贴到配置框中：

```json
{
  "registry-mirrors": [
    "https://docker.m.daocloud.io",
    "https://docker.1panel.live"
  ],
  "builder": {
    "gc": {
      "defaultKeepStorage": "20GB",
      "enabled": true
    }
  },
  "experimental": false
}
```

5. 点击 **Apply & Restart**

#### 可用的镜像加速器地址（2026 年更新）：

- DaoCloud: `https://docker.m.daocloud.io` ⭐⭐⭐⭐⭐
- 1Panel: `https://docker.1panel.live` ⭐⭐⭐⭐⭐
- 阿里云（需注册）: `https://<your-code>.mirror.aliyuncs.com`

### 方案二：使用官方镜像 + 镜像加速器

已更新配置文件使用官方镜像，配合镜像加速器使用：

```bash
# 使用更新后的配置文件
docker compose -f docker-compose.cn-fast.yml up -d --build
```

### 方案三：手动拉取镜像

如果镜像加速器配置后仍然很慢，可以手动拉取：

```bash
# 拉取 MySQL 官方镜像
docker pull mysql:8.0

# 拉取 Node.js 镜像
docker pull node:20-alpine

# 拉取 Bun 镜像
docker pull oven/bun:1

# 拉取 Nginx 镜像
docker pull nginx:1.25-alpine

# 然后重新构建
docker compose -f docker-compose.cn-fast.yml up -d --build
```


1. Docker Desktop → Settings → Docker Engine

### 步骤 2：验证配置

```bash
docker info | findstr "Registry Mirrors"
```

应该能看到配置的镜像加速器地址。

### 步骤 3：重新构建

```bash
# 停止当前构建
docker compose -f docker-compose.cn-fast.yml down

# 清理缓存
docker system prune -f

# 重新构建
docker compose -f docker-compose.cn-fast.yml up -d --build
```

## 🆘 常见问题

### Q1: 配置镜像加速器后仍然很慢？

**A**: 尝试以下步骤：
1. 重启 Docker Desktop
2. 更换镜像加速器地址（DaoCloud 或 1Panel）
3. 手动拉取镜像

### Q2: 提示 "pull access denied"？

**A**: 使用官方镜像，不要使用第三方仓库：
```bash
# 错误：registry.cn-hangzhou.aliyuncs.com/library/mysql:8.0
# 正确：mysql:8.0
```

### Q3: 构建卡在某个步骤？

**A**: 查看详细日志：
```bash
docker compose -f docker-compose.cn-fast.yml build --progress=plain
```

### Q4: 如何完全清理重新构建？

**A**: 
```bash
# 停止所有容器
docker compose -f docker-compose.cn-fast.yml down

# 清理未使用的镜像
docker image prune -a

# 清理构建缓存
docker builder prune -a

# 重新构建
docker compose -f docker-compose.cn-fast.yml up -d --build --force-recreate
```
