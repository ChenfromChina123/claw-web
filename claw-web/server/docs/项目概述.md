## claw-web/server 后端项目报告

---

### 一、项目概述

这是一个基于 **Bun + TypeScript** 构建的全栈 AI 编程助手后端服务，采用 **Master-Worker 容器编排架构**，支持多用户并发会话、WebSocket 实时通信、AI Agent 对话等功能。

---

### 二、技术栈

| 分类 | 技术选型 |
|------|---------|
| **运行时** | Bun (Node.js 高性能替代品) |
| **语言** | TypeScript |
| **HTTP服务器** | Bun.serve (内置高性能服务器) |
| **WebSocket** | Bun 内置 WebSocket |
| **数据库** | MySQL + mysql2 连接池 |
| **认证** | JWT (jose库) + bcryptjs 密码加密 |
| **AI接口** | Anthropic Claude / 通义千问 Qwen |
| **邮件** | nodemailer |
| **容器** | Docker + 进程管理 supervisord |

### 四、核心功能模块

#### 4.1 认证与授权

| 模块 | 功能说明 |
|------|---------|
| **JWT认证** | 通过 `jose` 库实现 token 生成与验证 |
| **密码安全** | `bcryptjs` 加密存储用户密码 |
| **GitHub OAuth** | 支持 GitHub 第三方登录 |
| **邮件验证** | 注册/找回密码的邮箱验证码 |
| **RBAC权限** | 基于角色的访问控制（user/admin等级） |
| **资源配额** | 按用户等级限制资源使用量 |

#### 4.2 会话管理

| 模块 | 功能说明 |
|------|---------|
| **会话CRUD** | 创建、加载、删除、重命名会话 |
| **消息持久化** | 自动保存用户消息与AI响应 |
| **会话共享** | 支持分享会话给其他用户 |
| **已打开文件** | 追踪会话中打开的文件列表 |
| **消息搜索** | 全会话历史搜索 |
| **自动标题生成** | AI根据对话内容生成会话标题 |

#### 4.3 Agent对话引擎

**核心文件：** `services/conversation/sessionConversationManager.ts`

**工作流程：**
```
用户消息 → 上下文构建 → AI模型调用 → 工具执行 → 结果反馈 → 流式响应
```

**关键特性：**
- 流式响应：AI生成内容实时推送
- 工具调用：自动识别并执行工具
- 上下文管理：自动注入规则和会话历史
- 中断机制：支持用户中止生成
- 多轮对话：完整的对话历史管理

#### 4.4 容器编排

**Master-Worker 分离架构：**

| 角色 | 职责 |
|------|------|
| **Master** | 所有认证、所有路由、数据库操作、用户管理 |
| **Worker** | Agent执行、命令运行、文件操作（无认证） |

**热池机制：**
- 预启动容器池，减少冷启动延迟
- 动态分配用户到可用容器
- 空闲容器自动回收
- 容器健康检查与自动恢复

**调度策略：** `schedulingPolicy.ts`
- 用户优先策略
- 轮询分配策略
- 加权随机策略

#### 4.5 工具系统

| 工具类型 | 描述 |
|---------|------|
| **文件操作** | 读取、写入、删除、移动文件 |
| **命令执行** | 在容器中执行shell命令 |
| **工作目录** | 管理用户的工作空间 |
| **工具注册** | 动态注册新工具到系统 |

#### 4.6 WebSocket通信

**支持的实时消息：**

| 消息类型 | 功能 |
|---------|------|
| `ping/pong` | 心跳检测 |
| `login` | Token登录 |
| `create_session` | 创建会话 |
| `user_message` | 发送用户消息 |
| `interrupt_generation` | 中止AI生成 |
| `get_tools` | 获取可用工具 |
| `agents_orchestration_*` | 多Agent编排 |

#### 4.7 监控与诊断

| 模块 | 功能 |
|------|------|
| **性能指标** | CPU、内存、响应时间监控 |
| **告警系统** | 异常情况自动告警 |
| **健康检查** | 系统组件状态检测 |
| **审计日志** | 记录用户操作行为 |

#### 4.8 部署相关

| 模块 | 功能 |
|------|------|
| **项目部署** | 一键部署前端应用 |
| **域名服务** | 自定义域名绑定 |

### 六、关键设计模式

1. **服务分层架构**
   - Routes（路由层）→ Services（服务层）→ Repositories（仓储层）→ Database

2. **单例模式**
   - SessionManager、ContainerOrchestrator 等核心组件使用单例

3. **Master-Worker架构**
   - 认证与业务逻辑分离，提高安全性与扩展性

4. **热池预热**
   - 容器池预启动减少响应延迟

5. **流式响应**
   - AI生成内容实时推送，前端即时展示
