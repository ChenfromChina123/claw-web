## 一、项目概述

### 1.1 项目名称
**Claude Code HAHA** - 基于 Claude Code 的本地化 Web 可视化 AI 编程助手

### 1.2 项目简介
Claude Code HAHA 是一个**全栈 AI 协作平台**，集成了 AI 对话、多智能体协作、工具执行、会话管理等核心功能。该项目基于 Anthropic 的 Claude Code 进行深度定制和本地化开发，旨在为开发者提供一个强大且易用的 AI 辅助编程环境。

### 1.3 核心价值主张
- **多模型支持**: 集成 Anthropic Claude 和阿里通义千问等多种 AI 模型
- **多 Agent 协作**: 实现主从式多智能体架构，支持任务自动分解和并行执行
- **丰富的工具集**: 内置 22+ 种工具，覆盖文件操作、Shell 命令、网络搜索等场景
- **Web 可视化界面**: 提供现代化的 Vue 3 前端，支持实时通信和状态监控
- **完善的权限系统**: 5 种权限模式，确保 AI 操作安全可控

### 2.1 主要用途

#### ✅ 核心使用场景

| 场景 | 描述 |
|------|------|

#### 4.1.3 会话管理器 (SessionManager)

**位置**: [server/src/services/sessionManager.ts](file:///d:/Users/Administrator/AistudyProject/HAHA/claude-code-haha/server/src/services/sessionManager.ts)

**核心职责**:
- 会话生命周期管理（创建/加载/保存/删除）
- 内存缓存优化（活跃会话驻留内存）
- 防抖持久化（2 秒防抖自动保存到数据库）
- 用户会话列表维护
- 会话标题自动生成（LLM 驱动）

**关键特性**:
```typescript
class SessionManager {
  // 单例模式
  static getInstance(): SessionManager
  
  // 内存缓存
  private sessions: Map<string, InMemorySession>
  
  // 防抖保存
  private saveDebounceTimers: Map<string, NodeJS.Timeout>
  
  // 核心方法
  async createSession(userId, title?, model?, force?): Promise<Session>
  async loadSession(sessionId): Promise<InMemorySession>
  async getUserSessions(userId): Promise<Session[]>

}
```

#### 4.3.3 核心执行流程

**入口**: [server/src/tools/agentTool.ts](file:///d:/Users/Administrator/AistudyProject/HAHA/claude-code-haha/server/src/tools/agentTool.ts)

```typescript
// Agent 执行流程
async function executeAgent(params: AgentParams): Promise<AgentResult> {
  // 1. 参数解析与校验
  const input = validateAndParse(params)
  
  // 2. 权限前置检查
  await checkPermissions(input, context)
  
  // 3. Agent 选择与路由
  const agent = agentRouter.route(input.subagent_type, input.name, input.team_name)
  
  // 4. 创建运行时上下文
  const runtimeContext = await createRuntimeContext(agent, params)
  
  // 5. 决定执行模式（同步/异步）
  if (input.run_in_background) {
    return launchBackgroundAgent(runtimeContext)
  }
  
  // 6. 启动 Agent 执行
  const result = await runAgent({
    agentDefinition: agent,
    promptMessages: buildPrompt(input.prompt),
    toolUseContext: runtimeContext,
    canUseTool: runtimeContext.toolPermission,
    maxTurns: input.max_turns || 100
  })
  
  // 7. 清理资源
  await cleanupResources(runtimeContext)
  
  return result
}
```

## 附录 C: 常见问题 FAQ

### Q1: 这个项目和原版 Claude Code 有什么区别？

**A**: 
- 原版 Claude Code 是 Anthropic 官方的 CLI 工具（终端命令行）
- 本项目是基于其开源代码进行**深度定制和二次开发**的 Web 版本
- 新增了：Web UI、多用户系统、多 Agent 协作、MySQL 持久化、权限系统等企业级功能

### Q2: 支持哪些 AI 模型？

**A**: 
- **Anthropic 系列**: Claude 3.5 Sonnet, Claude 3 Opus
- **阿里云系列**: 通义千问 Plus/Turbo/Max
- 可通过配置轻松扩展其他兼容 OpenAI API 格式的模型

### Q3: 如何添加新的自定义工具？

**A**: 
1. 在 `server/src/integration/enhancedToolExecutor.ts` 添加工具处理器
2. 在 `server/src/integrations/toolRegistry.ts` 注册工具
3. （可选）添加工具别名到 `server/src/tools/toolAliases.ts`
4. 编写单元测试
5. 前端 `ToolPanel.vue` 会自动展示新工具

### Q4: 数据存储在哪里？

**A**: 
- **用户/会话/消息**: MySQL 数据库
- **活跃会话**: 服务器内存（Map 缓存，带防抖持久化）
- **文件操作**: 本地文件系统（受路径访问控制）
- **工具执行历史**: MySQL tool_calls 表

### Q5: 如何保证 AI 操作的安全性？

**A**: 
- **五级权限模式**: 从完全信任到强制审批
- **工具黑白名单**: 精确控制可用工具集
- **路径访问控制**: 禁止访问敏感目录（如 System32）
- **危险命令检测**: `rm -rf /` 等命令需二次确认
- **最大轮次限制**: 防止无限循环消耗 Token
- **用户中断**: ESC 键随时中止执行
